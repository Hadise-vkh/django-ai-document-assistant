# Document Question Answering System

A document-based Question Answering system built with Django, Django REST Framework, LangChain, ChromaDB and OpenRouter.

The system allows users to upload DOCX documents, store their contents, ask questions about uploaded documents and receive answers generated by a Large Language Model (LLM) using Retrieval-Augmented Generation (RAG).

---

# Features

* Upload DOCX documents
* Extract and store document content
* Manage documents through Django Admin
* Manage documents through REST API
* Semantic search using embeddings
* Retrieval-Augmented Generation (RAG)
* Ask questions based on uploaded documents
* Generate answers using OpenRouter LLMs
* Store question and answer history
* Dockerized deployment

---

# Project Architecture

1. User uploads a DOCX document.
2. The system extracts the document text.
3. The content is stored in the database.
4. Embeddings are generated and stored in ChromaDB.
5. User submits a question.
6. Relevant document content is retrieved using semantic search.
7. LangChain builds the context.
8. The LLM generates an answer.
9. Question and answer are saved in history.

```bash

cd django-ai-project
```

---

# Environment Variables

Create a `.env` file in the project root:

```env
OPENROUTER_API_KEY=your_api_key_here
```

---

# Running With Docker

Build and start the application:

```bash
docker-compose up --build
```

Application URL:

```text
http://localhost:8000
```

Admin Panel:

```text
http://localhost:8000/admin
```

# API Endpoints

## Documents

### List Documents

```http
GET /api/documents/
```

### Upload Document

```http
POST /api/documents/
```

### Retrieve Document

```http
GET /api/documents/<id>/
```

### Update Document

```http
PUT /api/documents/<id>/
```

### Delete Document

```http
DELETE /api/documents/<id>/
```

See API.md for detailed documentation.
